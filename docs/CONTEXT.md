## 1. ИНФРАСТРУКТУРА

### Хранение и доставка медиа

| Тип | Хранилище | URL | Примечание |
|---|---|---|---|
| Финальные видео | Cloudflare R2 | media.lefiro.co | CF cache Edge TTL 1 мес, GET→HIT |
| Temp/клипы/аудио/картинки (VGF) | Backblaze B2 | videos.lefiro.co | Старые B2-URL живы, миграции нет |
| Инструментальное аудио / субтитры | Supabase Storage | — | Private bucket, signed URL |

- `uploadVideoToR2`: streaming через `createReadStream` (НЕ `readFileSync` — OOM на гигабайтных файлах), SigV4, endpoint standard, 1 retry, fallback → B2 + `[non-critical]`-алерт в ТГ.
- Cache Rule CF: `media R2 cache` (Hostname = media.lefiro.co → Edge TTL 1 мес). Файлы >512 МБ CF не кэширует на Free-плане (отдача из R2 напрямую, egress $0).
- Старый вердикт «WAF блокирует Railway → R2 навсегда» опровергнут живым ретестом: TLSv1.3 с Railway работает (была TLS-болячка per-account cert, прошла).
- R2 env Railway: `R2_ACCOUNT_ID / R2_ACCESS_KEY_ID / R2_SECRET_ACCESS_KEY / R2_BUCKET / R2_PUBLIC_BASE`.

### Устойчивость рендера (video-server, SHA ea7b622+)

- **startup-recovery**: при старте контейнера → все processing-job'ы → `failed` с причиной `orphaned by container restart (phase: X)` + идемпотентный refund + ТГ-алерт. Проверено боем и симуляцией.
- **SIGTERM-хендлер**: аннотирует активные job'ы (best-effort).
- **Heartbeat**: каждая волна клипов Stage 2 + каждый batch/merge concat/xfade. В Phase 5: `onPoll → makeHeartbeat(projectId)`, троттлинг 30с — нужен, чтобы watchdog не убивал ретрай burn-in (worst-case Phase 5 = 60 мин > watchdog 40 мин).
- **WATCHDOG_DRY_RUN=false**: порог 40 мин без прогресса → failed + refund + ТГ-алерт.
- **recoverOrphanedAudioJobs()**: pending/processing audio_jobs >15 мин → failed + refund.

### Деплой

- **Railway Watch Paths = `video-server/**`**: пуши, не трогающие video-server, дают статус SKIPPED. ⚠️ Push в video-server = рестарт контейнера = расстрел активных рендеров (см. WORKFLOW правило 2).
- **Vercel**: триггерится на любой push в main. `VERCEL_TOKEN` доступен в Railway env.
- **Pre-push hook**: `node --check video-server/index.js` — ⚠️ живёт только в `.git/hooks` (умрёт при переклонировании). Задача: перенести в `scripts/` + `core.hooksPath`.
- **UptimeRobot**: HTTP-монитор на /health, 5 мин, email-алерт.

## 2. КРЕДИТНАЯ И ПОДПИСОЧНАЯ СИСТЕМА (миграции 001–003)

### Два кошелька

| Кошелёк | Поле БД | Источник | Поведение |
|---|---|---|---|
| Тарифный | `plan_credits` | активация тарифа | Сгорает при истечении срока |
| Вечный | `purchased_credits` | топапы, free-стартовые, возвраты, компенсации | Не сгорает никогда |

- `credits` = материализованная сумма (`plan_credits + purchased_credits`), обновляется **только через rpc** (никаких прямых UPDATE на поле `credits`).
- `spend_credits` / `deduct_credits`: атомарны, сначала списываются тарифные, потом вечные, отказ без изменений при нехватке.
- `add_plan_credits`: кап `PLAN_MAX_CREDITS` на начисление (баланс не режется постфактум).
- Балансы до миграции 001 → мигрированы в вечный кошелёк.

### Жизненный цикл тарифа

1. **Активация**: `activatePlan(userId, plan, source)` — единая точка для ВСЕХ каналов (Paddle, ЮKassa, ТГ-бот, ручная). Добавляет тарифные кредиты нового плана, продлевает `plan_expires_at = max(now, expires) + 30д`. `chat_id` сохраняется при ТГ-активации.
2. **Продление при активном плане**: тарифные кредиты старого плана сохраняются (суммируются с новыми), срок считается от max.
3. **Понижение тарифа разрешено** (решение 22.07): покупка младшего при активном старшем — кредиты суммируются, фич-гейтинг ничего не отнимает, срок = max(now, expires)+30д.
4. **Истечение**: `rpc expire_plan` (идемпотентный: already_free/not_expired) — тарифные сгорают с транзакцией `plan_expired` (wallet='plan'), план → free. `plan_expires_at` сохраняется для аудита.
5. Не продлил → тарифные сгорают, аналитика закрывается, вечные кредиты остаются.
6. Позднее продление НЕ воскрешает сгоревшее.
7. `extend_plan(userId, days, reason, actor)` — компенсации с журналом `plan_events`. Гварды: plan_is_free / no_expiry_set (NULL-expires — вне модели, не продлевается).

### Кроны

- **9:00 UTC ежедневно**: `expire_plan` для истёкших. Защита: >20% платных истекло разом → ABORT + алерт (страховка от битых дат).
- **Рассылка −1..−3 дня**: ТГ по `chat_id` + email через Resend (graceful skip без ключа). Антиспам: `last_expiry_notice_at` 48ч.
- **Уведомление о сгорании**: ТГ-алерт юзеру в момент expire.
- **Retention 7:00 UTC ежедневно**: уборка медиа (DRY_RUN=false с 20.07.2026).

### Гейт аналитики

- `analytics-gate.ts` проверяет только `plan === 'free'` → 403 `plan_required` + UI-заглушка `PlanRequiredBlock`.
- Между paid-планами (basic/starter/pro/agency) функциональных различий НЕТ — один и тот же гейт.
- Карта серверного гейта: 11 роутов — niche, trends, channel, niche-finder, keywords, channel-plan, comments, channels-compare, rising-stars, revenue, reports.
- Инструменты-генераторы гейту аналитики **не подлежат**.

## 5. ИНСТРУМЕНТЫ

### Архитектурный принцип (волна 1+)

**Второй вход, не второй пайплайн.** Инструменты переиспользуют существующие API-роуты, модели, компоненты (эталон: `ScriptSettingsForm` — общий для Шага 1 студии и инструмента «Сценарий»).

- Результаты инструментов = `type='tool_run'` в таблице `projects`, `status=completed/failed`.
- Retention 72ч покрывает автоматически.
- Гейт аналитики к инструментам **не применяется** (generation-класс).
- `tool_run` — гость в студийном пайплайне: при переиспользовании пути студии проверять `type` перед достройкой пайплайна (прецедент: tool_run-озвучка получала `status='generating_subtitles'`).

### Волна 1 (7 инструментов, все в проде)

1. **Сценарий для видео** — полный паритет настроек Шага 1 студии (модель, стиль, тон, аудитория, хук, CTA, маркеры, паузы); опц. план (400 кр, двухфазно: план отображается и редактируем → затем скрипт); «Использовать в студии» создаёт `type='project'` → редирект на Шаг 3.
2. **SEO-пакет для ролика** — скрипт+тема → заголовок/описание/теги (750 кр).
3. **Посты из сценария** — 3 НЕПОХОЖИХ формата (ТГ-пост / Дзен-статья / тред) за один запрос, анти-шаблонность в промпте (700 кр).
4. **Уникализатор текста** — повышение уникальности / убрать следы ИИ / оба (660/660/1320 кр). Де-факто переводчик (EN-вход + RU-выход), отдельную «Перевести» не делали.
5. **Озвучка текста** — все 6 движков (sync: elevenlabs/openai/google/apihost; async: voicer/secretvoicer через Railway). `pollId` в localStorage (не теряется при закрытии вкладки).

### Волна 2 — Субтитры по аудио (в проде, SHA 442b63f+)

- Движок: OpenAI Whisper через Railway `/transcribe`. Лимит **25 МБ на файл** (~20–30 мин речи).
- Форматы: mp3/m4a/aac/ogg/wav. Языки: SCRIPT_LANGUAGES (28) + «Авто».
- Первый файловый вход в проекте: `type='tool_audio'` в `/api/upload/sign` (путь `{userId}/tool/{uuid}.ext`, без project_id), валидация MIME + расширение + размер.

- Входной аудиофайл удаляется из Storage после успешного сохранения.
- ⚠️ **Паттерн-ловушка**: `createSignedUrl` вызывается ПОСЛЕ загрузки файла в Storage (в `generate/subtitles` по `storage_path`), не до неё (Supabase: «Object not found»).

## 7. КЛЮЧЕВЫЕ ДИАГНОЗЫ И ФИКСЫ

### libass — Phase 5 retry (SHA 7c223cb + 0886d63)

«Separator is found, but chunk is longer than limit» — **транзиентная ошибка VGF**, не детерминированный баг контента. Доказано бисекцией: полный боевой .ass (1159 блоков, 145 КБ, кириллица до 161 байт/поле) на 58-мин видео 1280×720 — succeeded (VGF job 70f52831, 4.2 мин). Порогов по числу блоков/размеру/длительности НЕТ.

Фикс:
- Phase 5: 2 попытки burn-in, пауза 10с.
- Heartbeat в Phase 5 (troтлинг 30с) — без него watchdog убивал бы ретрай.
- Честная деградация при двойном фейле: `error_message='[warn:subtitles_burn_failed:...]'` (единственный источник сигнала) → `subtitle_warn:true` → янтарный баннер в Step6Video. `warnings jsonb` — аудит-дубль. ТГ 🟡 degraded (не 🔴 и не [non-critical]).
- Refund за burn-in не делается — оплата была на шаге транскрипции, переиспользуется.

### NSFW-фильтр fal (SHA cfaf65b)

fal при срабатывании safety checker заменяет изображение **чёрным кадром той же размерности** + `has_nsfw_concepts: [bool]`. Ранее: URL валидный → списание без картинки (включая студийный пайплайн часовых видео).

Фикс: **6 генераторов** (`images/route.ts` × 3 + `image-single/route.ts` × 3): читаем `has_nsfw_concepts`, флагнутая картинка не оплачивается, 1 авторетрай, при двойном блоке — янтарный щит + ↺ в UI. Safety checker **оставлен включённым** (оферта §6.2.3). Ложные срабатывания: фитнес/мускулатура/анатомия.

### content[0] — класс на 19 мест (SHA e0de73a)

Claude Sonnet 5 с extended thinking возвращает `{type:'thinking'}` первым блоком `content`. Код, берущий `content[0]`, получал не text-блок — «Ошибка анализа».

Фикс: `.find(b => b.type === 'text')` во всех 10 analytics-роутах (19 мест). Generate-роуты были безопасны. Класс найден через grep, закрыт до эпидемии.

### ЮKassa webhook: body.event, не body.type (SHA 4587d78)

`body.type` всегда `"notification"` — бесполезное поле. Actionable-поле: `body.event === 'payment.succeeded'`. Ошибка стояла с начала и была источником молчаливых ack всех вебхуков без активации.

## 8. ГЕНЕРАЦИЯ СЦЕН

- `CLAUDE_CHUNK=50`, `max_tokens=min(64000, max(4000, chunk×150))`. Лимит Haiku 4.5 = **64K output** (не 8192 — фантом опровергнут документацией).
- Client timeout 120с, worst-case 245с < 300с maxDuration.
- `SCENES_SYSTEM_PROMPT`: нормы против символов/монтажных указаний/«indicated by» + says/asks → визуальное действие.
- `sanitizeScenePrompt`: словарная замена триггеров на основном и fallback-путях.
- Fallback-шаблон картинки БЕЗ темы дословно (schnell рисует любые слова вывеской — живой факт: 75 кадров с надписью темы).
- `fallbackTopic`: `user_topic → title → script_head(8 слов)` (отдельный параметр).
- Алерт `scenes_fallback` при доле fallback >10% + причина (stop_reason/статус) в логе.
