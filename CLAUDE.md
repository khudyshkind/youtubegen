# CLAUDE.md — Правила для Claude Code

## Язык общения
Всегда отвечай пользователю на русском языке.
Комментарии в коде пиши на английском, все сообщения пользователю — на русском.

## Ритуал начала каждой сессии
Перед тем как что-либо делать выполни по порядку:
1. Прочитай docs/MEMORY.md — контекст проекта и что уже сделано
2. Прочитай docs/ERROR_LOG.md — ошибки которые уже решались
3. Прочитай docs/PROGRESS.md — статус всех задач
Сообщи пользователю кратко: статус проекта и что планируешь делать.

## Ритуал завершения после каждой задачи
1. Обнови docs/MEMORY.md — что было сделано
2. Обнови docs/PROGRESS.md — отметь выполненные пункты
3. Если исправлял ошибку — запиши в docs/ERROR_LOG.md
4. Сделай git commit

## При возникновении ошибки
1. Сначала проверь docs/ERROR_LOG.md — возможно ошибка уже решалась
2. Если нашёл решение — применяй сразу
3. После нового решения — обязательно запиши в ERROR_LOG

## Структура файлов
Все файлы строго по структуре:
- src/app/ — страницы и API routes
- src/components/ — React компоненты  
- src/lib/ — утилиты, типы, клиенты
- supabase/ — схема базы данных
- docs/ — только документация
- tmp/ — временные файлы (в .gitignore)

## Кодовые стандарты
- TypeScript strict, никогда не использовать any
- Всегда try/catch в API routes
- Никогда не хардкодить ключи — только через process.env
- Если добавляешь env переменную — сразу добавь в .env.example

## Supabase
- В API routes: createServerSupabase() или createServiceClient()
- В Client Components: только createClient()
- Никогда не использовать service role key на клиенте

## Кредиты
- Перед каждым вызовом платного API — проверить кредиты через requireCredits()
- Если NOT_ENOUGH_CREDITS — вернуть { ok: false, code: 'NO_CREDITS' } со статусом 402

## Git
- Коммит после каждой завершённой фичи
- Формат: feat: добавить генерацию субтитров
- Никогда не коммитить .env.local

## Перед тем как сказать готово
- Убедиться что код компилируется: npx tsc --noEmit
- Обновить docs/MEMORY.md и docs/PROGRESS.md
- Сделать коммит

## Инфраструктура деплоя

### Railway (video-server)
- Сервис: `ytgen-video-server` (serviceId `75bf2476-6da9-48d2-a3f1-7f37313295a7`)
- Watch Paths: `["video-server/**"]` — Railway деплоит только при изменениях в папке `video-server/`
- Настроено через GraphQL: `serviceInstanceUpdate(watchPatterns: ["video-server/**"])`
- Изменения в `src/`, `supabase/`, конфигах корня → Railway пропускает (статус `SKIPPED`)

### Vercel (веб-часть)
- Триггерится на любой push в `main`
- Папка сборки: корень проекта (`src/`, `public/`, конфиги Next.js)
- Vercel игнорирует `video-server/` автоматически (не входит в билд Next.js)

## О проекте
Этот проект — YouTubeGen, SaaS для автоматизации YouTube-контента.
Пайплайн: Сценарий (Claude API) → Озвучка (Eleven Labs) → Субтитры (Whisper) → Иллюстрации (Flux) → Видео (FFmpeg) + SEO (Claude API).
Основная аудитория — русскоязычные YouTube блогеры.
Все тексты интерфейса на русском языке.
